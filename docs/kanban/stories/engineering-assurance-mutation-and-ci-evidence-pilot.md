# ENG-017J: Pilot mutation checks and CI assurance evidence

## Intent

Two closing moves for the lane. First, test the tests: pilot a mutation
workflow that proves the suite notices when enforcement disappears — the
mutant that must die is "the validation call was removed and invalid data
persisted." Second, make CI the author of the assurance record: a
revision-bound evidence artifact that replaces agent-authored "all checks
pass" claims.

## Decision context

Implements `docs/designs/verification-architecture.md` §§ "Mutation
strategy" and "CI evidence artifact" under ADR-004 decisions 5 and 7
("CI-generated evidence is authoritative over agent-authored completion
claims"). The 2026-07-12 audit is the motivating record: 7 done cards
demoted, 17 done cards with no completion evidence at all — self-attested
status is the failure mode this card retires.

## Scope

- Mutation pilot scoped to boundary-critical namespaces only:
  `law.operations`, `application.validation`, and the ENG-017D law mappings.
  Candidate tools per the research trail (Mutant — Leiningen-oriented, likely
  reject; Heretic — verify deps.edn compatibility): a one-week proof per the
  evaluation protocol in inbox `06.19.25`.
- Hard gate for adoption: a deliberately removed `validate!` call and a
  wrong-schema-selection mutant must both be detected, and the tool must run
  under `clojure -M`. Otherwise: record the rejection decision with basis and
  implement the two critical mutants as hand-rolled sabotage tests instead.
- Surviving-mutant classification vocabulary: test gap / equivalent /
  irrelevant / tool limitation — reviewed, never silently ignored.
- CI evidence artifact (EDN) per checked revision: commands + exits, property
  seeds, adapter suites run, mutation outcomes, interop delta vs. baseline —
  generated by CI, uploaded as a build artifact.
- Specify protected/rotated adversarial-case handling (ADR-004 §8) without
  exposing fixtures to untrusted PR contexts — specification and location,
  full private-suite build-out is follow-up.

## Non-goals

- No whole-repo mutation score or coverage percentage gate (explicitly
  rejected by ADR-004). No board-transition enforcement (separate decision
  candidate from the audit). No new laws (ENG-017I).

## Invariants

- The evidence artifact is generated by CI execution, never authored or
  edited by an agent or human.
- A meaningful surviving mutant blocks adoption sign-off until classified.

## Verification

| Claim | Evidence | Location |
|---|---|---|
| Removed-validation mutant dies | Pilot run report (or sabotage test red) | mutation job / sabotage test ns |
| Wrong-schema mutant dies | same | same |
| Artifact reproducible + revision-bound | Two CI runs at same rev produce equivalent artifacts (modulo timestamps) | CI artifact |
| Artifact records all required fields | Schema-validated artifact (dogfood: validate it with Malli) | artifact schema test |

## Acceptance criteria

- Tool adoption or rejection is a recorded decision with the evaluation
  evidence; either way, the two critical mutants are provably detected.
- CI produces and uploads the evidence artifact on every merge-path run;
  its schema is registered in law.
- Private/rotated case handling is specified in a card comment or linked
  design note.

## Dependencies and interfaces

- Depends on ENG-017H (static gates + interop baseline feed the artifact)
  and ENG-017I (properties/seeds it records).
- Provides to the board process: the machine-checkable completion-evidence
  source the audit's transition-enforcement proposal can consume.

## Risks and open questions

- Clojure mutation tooling maturity is the known risk (hence `risk: high`
  and the built-in rejection path with sabotage tests as the fallback that
  still satisfies the intent).

## Completion evidence

Pilot evaluation notes, adoption/rejection decision, a linked CI run with
its artifact, reviewer named at done.

## Would have gated

The audit's central finding: every demoted card claimed done on self-attested
or absent evidence. With the CI artifact required at review→done, "done with
no evidence" stops being representable.
