## Immediate amendment

First, add this to `STYLE.md`:

```md
## The form is valid before it compiles

The construction form is valid design data whether or not every symbol is
implemented as executable Clojure today. It declares a target shape against
which existing code can be read.

Current source may partially instantiate the form, contain older shapes, or
contradict a current boundary law. Each difference is an architectural
observation: a candidate for discovery, design, and bounded remediation‚ÄĒnot a
reason to weaken the form.

Do not perform a repository-wide namespace shuffle merely to make paths appear
conformant. Realize the form through reviewable changes that state the current
gap, intended layer boundary, contracts introduced, compatibility plan, and
verification evidence.
```

That corrects the important framing failure: the kernel is a **future-readable data structure**, not a compliance claim about the current repository.

## Then: document governance

Create:

```text
docs/process/document-governance.md
```

This is now the right next policy because agents need concrete constraints for writing the documents that carry ő∑, őľ, ő†, research, plans, decisions, and acceptance.

It should define:


| Concern | Policy decision |
| :-- | :-- |
| Document kinds | Research brief, source record, finding, research report, design, ADR, process policy, epic, story, completion/review record |
| Document status | `exploratory`, `draft`, `review`, `active`/`accepted`, `superseded`, `retired`, `archived` |
| Required relations | Which artifacts may inform, implement, supersede, or accept others |
| Templates | Required frontmatter and headings per kind |
| Examples | Curated good examples plus deliberately invalid anti-examples |
| Checker | Structural errors, relation errors, and semantic-review warnings |
| Migration | How old docs gain status/classification without rewriting history |
| Exceptions | How to create an intentional nonstandard document without silently creating a new kind |
