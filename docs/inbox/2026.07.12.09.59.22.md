@GitHub

I’ve sketched an Eta-Mu-Sol–style Clojure project that treats your markdown notes as typed objects with contracts, extracts headings/links/terms into a graph and glossary, and writes a machine-usable index that your agents can consume as evidence instead of raw text piles. The whole project is packaged as a `deps.edn` zip you can download, inspect, and evolve.

## Strategic frame: from folders to geometry

The project is designed as a pre-agent “notes geometry” layer:

- **Domain**: what a note, glossary entry, and note-graph *mean* (no I/O).
- **Shape**: how markdown is parsed into those typed shapes (pure transforms).
- **Law/contract**: Malli schemas and guards that decide what’s admissible (e.g. canonical note uniqueness).
- **Infra**: a simple FS scanner and index runner that you can point at a repo.

You get:

- A **typed `Note` object** per markdown file (meta, headings, links, terms).
- A **glossary entry model** for terms that appear across notes.
- A stubbed **graph model** for the relationships you care about.
- Contracts that enforce “multiple drafts allowed, but only one canonical per path”, so your speculative notes can coexist with a small set of authoritative specs.

You can run this against `docs/notes`, `docs/research`, `docs/designs` in `octave-commons/Truth` and siblings to build an evidence index that later Eta-Mu-Sol agents can query, cite, and reason over.

(ns eta-mu-sol.domain.note
  (:require [malli.core :as m]))

(def NoteId
  [:map
   [:repo string?]
   [:path string?]])

(def NoteMeta
  [:map
   [:id NoteId]
   [:title string?]
   [:date {:optional true} string?]
   [:categories {:optional true} [:set string?]]
   [:tags {:optional true} [:set string?]]
   [:canonical? {:optional true} boolean?]])

(def Heading
  [:map
   [:level int?]
   [:text string?]])

(def Link
  [:map
   [:target string?]
   [:kind keyword?]])

(def Note
  [:map
   [:meta NoteMeta]
   [:headings [:sequential Heading]]
   [:links {:optional true} [:sequential Link]]
   [:terms {:optional true} [:set string?]]])

(defn make-note-meta
  [{:keys [repo path title date categories tags canonical?]}]
  {:pre [(string? repo) (string? path) (string? title)]}
  {:id {:repo repo :path path}
   :title title
   :date date
   :categories (when (seq categories) (set categories))
   :tags (when (seq tags) (set tags))
   :canonical? (boolean canonical?)})

(defn valid-note? [note]
  (true? (m/validate Note note)))
```

(ns eta-mu-sol.domain.graph
  (:require [malli.core :as m]))

(def EdgeType
  [:enum :note->note :note->term :term->note :term->term])

(def Edge
  [:map
   [:from string?]
   [:to string?]
   [:type EdgeType]
   [:labels {:optional true} [:set keyword?]]])

(def Graph
  [:map
   [:nodes [:set string?]]
   [:edges [:sequential Edge]]])

(defn empty-graph []
  {:nodes #{} :edges []})

(defn add-node [g node]
  (update g :nodes (fnil conj #{}) node))

(defn add-edge [g {:keys [from to type labels]}]
  (update g :edges conj {:from from :to to :type type :labels (set labels)}))

(defn valid-graph? [g]
  (true? (m/validate Graph g)))

(ns eta-mu-sol.shape.markdown
  (:require [clojure.string :as str]))

(defn heading-line? [line]
  (boolean (re-matches #"^#+\s+.+$" line)))

(defn parse-heading [line]
  (let [hashes (take-while #(= % \#) line)
        level (count hashes)
        text (-> line (subs level) str/trim)]
    {:level level :text text}))

(defn extract-headings [markdown]
  (->> (str/split-lines markdown)
       (filter heading-line?)
       (map parse-heading)
       (into [])))

(def link-pattern #"\[([^\]]+)\]\(([^)]+)\)")

(defn extract-links [markdown]
  (->> (re-seq link-pattern markdown)
       (map (fn [[_ _target url]]
              {:target url
               :kind (cond
                       (str/starts-with? url "http") :external
                       (str/ends-with? url ".md") :note
                       :else :other)}))
       (into [])))

(def term-pattern #"\[\[([^\]]+)\]\]")

(defn extract-terms [markdown]
  (->> (re-seq term-pattern markdown)
       (map second)
       (map str/trim)
       (into #{})))

(defn parse-frontmatter [markdown]
  (let [lines (str/split-lines markdown)]
    (when (and (seq lines) (= "---" (first lines)))
      (let [rest-lines (next lines)
            fm-lines (take-while #(not= "---" %) rest-lines)]
        (->> fm-lines
             (map #(str/split % #":" 2))
             (keep (fn [[k v]]
                     (when (and k v)
                       [(keyword (str/trim k)) (str/trim v)])))
             (into {}))))))
```

(ns eta-mu-sol.shape.notes-index
  (:require [eta-mu-sol.domain.note :as note]
            [eta-mu-sol.shape.markdown :as md]))

(defn build-note
  [{:keys [repo path content]}]
  (let [front (md/parse-frontmatter content)
        headings (md/extract-headings content)
        links (md/extract-links content)
        terms (md/extract-terms content)
        title (or (:title front) (-> headings first :text) path)
        date (:date front)
        categories (some-> front :categories (clojure.string/split #",") (map clojure.string/trim))
        tags (some-> front :tags (clojure.string/split #",") (map clojure.string/trim))
        canonical? (= "true" (:canonical front))
        meta (note/make-note-meta {:repo repo
                                   :path path
                                   :title title
                                   :date date
                                   :categories categories
                                   :tags tags
                                   :canonical? canonical?})]
    {:meta meta
     :headings headings
     :links links
     :terms terms}))

(defn index-notes [note-inputs]
  (mapv build-note note-inputs))

(ns eta-mu-sol.law.note-contract
  (:require [malli.core :as m]
            [eta-mu-sol.domain.note :as note]))

(def note-validator (m/validator note/Note))

(defn ensure-note! [n]
  (when-not (note-validator n)
    (throw (ex-info "Invalid note" {:note n :explain (m/explain note/Note n)})))
  n)

(defn canonical-notes-by-path [notes]
  (->> notes
       (filter (comp true? :canonical? :meta))
       (group-by (comp :path :id :meta))))

(defn ensure-unique-canonical! [notes]
  (doseq [[path cn] (canonical-notes-by-path notes)]
    (when (> (count cn) 1)
      (throw (ex-info "Multiple canonical notes for path" {:path path :notes cn}))))
  notes)

(defn enforce-note-contracts! [notes]
  (doseq [n notes] (ensure-note! n))
  (ensure-unique-canonical! notes))
```

(ns eta-mu-sol.infra.fs-scan
  (:require [clojure.java.io :as io]))

(defn markdown-file? [f]
  (and (.isFile f)
       (.endsWith (.getName f) ".md")))

(defn scan-dir [root-dir]
  (let [root (io/file root-dir)]
    (when-not (.exists root)
      (throw (ex-info "Root directory does not exist" {:root root-dir})))
    (->> (file-seq root)
         (filter markdown-file?)
         (map (fn [f]
                {:repo (.getName (.getParentFile root))
                 :path (.getPath f)
                 :content (slurp f)})))))

(ns eta-mu-sol.infra.index-runner
  (:require [clojure.edn :as edn]
            [clojure.java.io :as io]
            [eta-mu-sol.infra.fs-scan :as fs]
            [eta-mu-sol.shape.notes-index :as idx]
            [eta-mu-sol.law.note-contract :as law-note]))

(defn build-index [{:keys [root-dir output-file]}]
  (let [notes (idx/index-notes (fs.scan-dir root-dir))
        _ (law-note/enforce-note-contracts! notes)]
    (spit output-file (pr-str notes))
    {:output-file output-file}))

(defn -main [& args]
  (let [root-dir (or (first args) ".")
        output-file (or (second args) "notes-index.edn")]
    (build-index {:root-dir root-dir :output-file output-file})))
```

## How to run this against your repos

1. **Unzip and inspect**
Download the `eta-mu-sol-notes-geometry.zip` artifact and unzip it into a working directory.
2. **Wire to a docs tree**
Point `build-index` at something like `../Truth/docs/notes` or a higher-level root:

```bash
cd eta-mu-sol-notes-geometry
clojure -M -m eta-mu-sol.infra.index-runner ../Truth/docs/notes notes-index.edn
```

3. **Use the index**
The resulting `notes-index.edn` is a vector of `Note` maps with:
    - `:meta` recording repo, path, title, categories/tags, canonical flag.
    - `:headings` giving a quick outline of each note.
    - `:links` enumerating internal/external links.
    - `:terms` capturing `[[Glossary Term]]` occurrences.

You can then:
    - Feed this into your **OpenPlanner** graphs as a structured layer instead of embedding-only clustering.
    - Build a **glossary generator** that walks `:terms` and produces authoritative definitions, then replace term occurrences in markdown with links to the glossary.
    - Attach **contracts** around “authoritative vs exploratory” notes via `:canonical?` and the `note_contract` namespace, so agents know which spec or research documents count as binding evidence.
4. **Extend within Eta-Mu-Sol constitution**
    - Keep new domain shapes under `domain/`.
    - Add new pure transforms (e.g. cross-repo graph construction, vocabulary clustering) under `shape/`.
    - Tighten admissibility rules (e.g. “every canonical note must cite at least N sources”) in `law/`.
    - Add Babashka tasks or bb-compatible runners under `infra/` (e.g. `bb notes:index`).

This gives you a starting, lawful Clojure system where your markdown is no longer just a directory tree but a typed, contracted geometry that agents—and you—can navigate, query, and summarize without losing the shape of your long-horizon thinking.

***

## Document lifecycle: states, not just locations

Think of each markdown file as having **state** independent of its directory:

**Origin axis**

- **External-origin note**
Created outside any explicit workflow (you wrote something, pasted an agent dump, etc.).
- **Process-origin artifact**
Created *as the effect of* a known process step (research pass, card breakdown, retro, etc.).

**Status axis**

- **Raw** – unstructured, not yet touched by the perception system.
- **Inbox** – acknowledged as “we don’t know where this belongs yet”; quarantined from being treated as evidence.
- **Structured** – parsed, labeled, and attached to at least one process or topic.
- **Canonical** – accepted as an authoritative spec/research/design for some scope.
- **Archived** – kept for provenance but no longer active in planning.

**Role axis**

- **Signal** – captures “something happened” or “someone thought X”, but is not itself a spec.
- **Evidence** – structured signal attached to a claim or hypothesis.
- **Spec/Design/Research** – process artifacts that encode decisions, models, or findings.

A lifecycle for markdown then looks roughly like:

1. `External raw note` → `Inbox note` (you or an agent drop it in explicitly as “undecided”).
2. `Inbox note` → `Structured signal doc` (perception system runs, extracts features, assigns minimal metadata).
3. `Structured signal doc` → `Evidence bundle` (linked to cards, hypotheses, research tasks).
4. `Evidence bundle` → `Spec/Design/Research` doc (when you commit to a decision or model).
5. Any state → `Archived` (explicit deactivation with provenance preserved).

Directories (`docs/notes`, `docs/research`, `docs/designs`, `docs/specs`) stay, but they become *views* over state, not the state itself.

***

## Inbox as an explicit contract, not a dumping ground

You already use an “inbox” as a psychological escape valve: “belongs where it is” vs “we don’t know where this belongs.” Formalize that as a **contract**:

- **Inbox state definition**
A file is “Inbox” if:
    - Its origin is external (not created by a known workflow step).
    - Its name or minimal parse fails to match any current process pattern.
